SC-51: HARDWARE-BASED PROTECTION

Control Description

  1. SC-51a. Employ hardware-based, write-protect for [Assignment: organization-defined system firmware components]; and
  2. SC-51b. Implement specific procedures for [Assignment: organization-defined authorized individuals] to manually disable hardware write-protect for firmware modifications and re-enable the write-protect prior to returning to operational mode.
