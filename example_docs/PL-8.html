PL-8: INFORMATION SECURITY ARCHITECTURE

Control Description

The organization:

  1. PL-8a. Develops an information security architecture for the information system that:
    1. PL-8a.1. Describes the overall philosophy, requirements, and approach to be taken with regard to protecting the confidentiality, integrity, and availability of organizational information;
    2. PL-8a.2. Describes how the information security architecture is integrated into and supports the enterprise architecture; and
    3. PL-8a.3. Describes any information security assumptions about, and dependencies on, external services;
  2. PL-8b. Reviews and updates the information security architecture [Assignment: organization-defined frequency] to reflect updates in the enterprise architecture; and
  3. PL-8c. Ensures that planned information security architecture changes are reflected in the security plan, the security Concept of Operations (CONOPS), and organizational procurements/acquisitions.

Control Enhancements

  • PL-8 (1) - DEFENSE-IN-DEPTH

    The organization designs its security architecture using a defense-in-depth approach that:

    1. PL-8 (1)(a) Allocates [Assignment: organization-defined security safeguards] to [Assignment: organization-defined locations and architectural layers]; and
    2. PL-8 (1)(b) Ensures that the allocated security safeguards operate in a coordinated and mutually reinforcing manner.

  • PL-8 (2) - SUPPLIER DIVERSITY

    The organization requires that [Assignment: organization-defined security safeguards] allocated to [Assignment: organization-defined locations and architectural layers] are obtained from different suppliers.

Baselines

MODERATE, HIGH
